# 整体设计

采用典型的编译器架构，包括：

1. 词法分析：将程序分割为若干Token

2. 语法分析：采用预读的递归下降翻译，将Token流与语法相匹配，建立语法树

3. 中间代码生成及优化：使用语法制导翻译，将语法树转为llvm中间代码表示，并进行多次优化遍扫描

4. 后端代码生成及优化：将llvm ir转为更接近底层的lir，进行图着色寄存器分配，在此基础上将每个中端指令翻译为后端指令并进行优化

# 中间代码设计

在中间代码生成与优化阶段，采用llvm ir，主要特性包括：

- 强类型：变量类型不能隐式转换，特别是Cmp指令的I1类型结果与I32间的转换，以及数组类型与指针类型的转换

- SSA：静态单赋值，有助于中间代码优化时的分析

- 函数与基本块：相较于后端的指令+标签结构，中间表示额外包括函数与基本块结构，程序的结构层次更规范，有利于优化时的调整

- use-def链：每个中间形式的value都记录其user，以便于优化时对指令的分析与替换



在中间代码生成后，引入更多底层细节，将中间表示翻译为类似llvm ir的底层表示形式，这一过程包括：

- Phi指令翻译为Move指令，这一过程会破坏move指令之外指令的SSA性质

- 为不能使用立即数的指令插入move指令，使虚拟寄存器替换立即数的位置

- 使用Push指令，在函数调用时使用栈传递高于参数寄存器数目的参数；使用GetParam指令，在函数使用相应形式参数时从栈对应位置中取出

- 使用图着色寄存器分配算法，为每个虚拟寄存器分配物理寄存器，此过程中会同时进行Move指令的合并与溢出寄存器的栈空间分配

# 代码优化

中间代码上的优化主要分为四类：

- 代数层面的冗余消除与强度削弱，包括常量折叠、乘除法优化、死代码删除、GVN-GCM、条件语句的传播等

- 函数相关，包括函数内联、尾递归消除

- 内存相关，包括mem2reg、仅使用部分元素的数组转为局部变量、以及对load-store指令的简单值编号替换、局部大数组提升为全局数组等

- 循环相关，包括循环展开、归纳变量强度削弱、去开关化、循环交换等



底层代码上的优化主要包括伸直化、窥孔优化等
