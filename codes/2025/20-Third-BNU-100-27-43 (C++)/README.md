## 1. 项目简介

本项目是一个功能完整的编译器，旨在将 SysY 语言（C语言的一个子集）的源程序编译为 RISC-V 64位汇编代码。该编译器实现了从源代码解析、语义分析、多层次优化到目标代码生成的完整流水线。

项目整体架构清晰，分为前端、中端和后端三个核心部分，采用了模块化的设计思想，易于理解和扩展。

## 2. 核心特性与技术栈

*   **前端 (Frontend)**
    *   **词法分析**: 使用 `Flex` (lex.yy.c) 进行词法分析，将源代码切分为Token流。
    *   **语法分析**: 使用 `Bison` (sysy.tab.c) 进行语法分析，构建抽象语法树 (AST)。
    *   **AST**: 设计了完整的AST数据结构 (ast.h, ast.c) 来表示程序的语法结构。
    *   **语义分析**: 实现了基于作用域的符号表管理 (symbol.h, symbol.c) 和递归下降的语义检查 (semantic.h, semantic.c)，包括类型检查、变量定义检查等。

*   **中端 (Middle-end)**
    *   **AST层优化**: 在生成IR之前，对AST进行初步优化，包括常量折叠和死代码消除 (ast\_optimizer.h, ast\_optimizer.cpp)。
    *   **中间表示 (IR)**: 自定义了一套类似 LLVM IR 的强类型中间表示 (IR.h)，包含完整的指令集、基本块、函数等结构，为后续优化提供了坚实基础。
    *   **IR生成**: 通过 `IRBuilder` (IRBuilder.h, IRBuilder.cpp) 将优化后的AST转换为自定义的IR。
    *   **IR层优化**: 实现了一个模块化的、基于Pass的优化器 (`IROptimizer`)，包含多个经典的、高级的编译优化算法。

*   **后端 (Backend)**
    *   **目标代码生成**: `AsmGenerator` (AsmGenerator.h, AsmGenerator.cpp) 负责将优化后的IR翻译为 RISC-V 64位汇编代码。
    *   **寄存器分配**: 实现了一个带有溢出(Spilling)机制的线性扫描风格寄存器分配器，能够有效管理有限的物理寄存器。
    *   **栈帧管理**: 自动进行函数栈帧布局，处理函数调用、参数传递和局部变量存储。

## 4. 模块功能详解

### 4.1 前端

前端负责将 SysY 源代码转化为内部的 AST 表示。

*   `lex.yy.c`: 定义了SysY语言的关键字、标识符、常量（整型、浮点型）、运算符和分隔符的词法规则。
*   `sysy.tab.c`: 定义了SysY的上下文无关文法，在解析过程中调用`ast.c`中的构造函数生成AST节点，构建完整的语法树。
*   `ast.h`/`ast.c`: 定义了`ASTNode`结构体，通过`union`和`enum`来区分不同类型的语法单元，如函数定义、二元运算、if语句等。
*   `symbol.h`/`symbol.c`: 实现了带作用域的符号表。`Scope`结构体通过父子关系形成作用域树，`Symbol`结构体存储变量、常量或函数的信息。
*   `semantic.c`: 遍历AST，利用符号表进行语义检查，如变量是否声明、类型是否匹配等。一个关键特性是，它会在AST中自动插入`NODE_TYPE_CAST`节点来处理int和float之间的隐式类型转换，简化了后续IR生成阶段的逻辑。

### 4.2 AST层优化器 (`ASTOptimizer`)

在生成IR之前，此模块直接对AST进行优化，旨在化简语法树，减少后续IR的复杂度。

*   **常量折叠 (Constant Folding)**: 递归地计算在编译期就能确定值的常量表达式，并用其结果替换掉原来的表达式节点。
*   **死代码消除 (Dead Code Elimination)**: 通过两遍扫描（第一遍分析变量使用情况，第二遍进行优化），移除从未被使用过的局部变量声明。
*   **分支优化**: 对`if`和`while`语句，如果其条件是常量，则可以直接确定分支走向，从而移除整个分支结构或将其转换为无条件跳转。

### 4.3 中间表示与IR生成 (`IR.h`, `IRBuilder`)

这是编译器的核心中枢。

*   `IR.h`: 定义了一套完整的、面向对象的中间表示。设计上参考了LLVM，包括`Value`, `Instruction`, `BasicBlock`, `Function`等核心类。指令集丰富，支持整数、浮点、内存访问、分支、函数调用等多种操作。
*   `IRBuilder.h`/`IRBuilder.cpp`: 这是一个Visitor模式的实现，它遍历优化后的AST，为每个AST节点生成对应的IR指令序列。它负责管理基本块的创建和连接、临时变量的生成，以及复杂结构（如数组初始化）的IR转换。

### 4.4 IR层优化器 (`IROptimizer`)

本编译器采用了基于Pass的优化框架。优化器会**循环执行**一系列优化Pass，直到IR不再发生变化，从而达到优化效果的最大化。

*   **Alloca提升 (Alloca Hoisting)**: 将函数内所有`alloca`指令（局部变量内存分配）统一移动到函数入口基本块的起始位置。这是一个重要的预处理步骤，为`Mem2Reg` Pass创造了条件。

*   **Mem2Reg**: 实现经典的`mem2reg`算法，将栈上的局部变量（由`alloca`指令分配）提升为SSA（静态单赋值）形式的虚拟寄存器。此Pass依赖于**控制流图（CFG）分析**和**支配树（Dominator Tree）与支配边界（Dominance Frontier）**的计算，能够正确地在需要的位置插入PHI指令，是现代编译优化的基石。

*   **函数内联 (Function Inlining)**: 这是一个强大的跨基本块优化。它会选择一些“小”函数（根据指令数量判断），将其函数体直接嵌入到调用点，从而消除函数调用的开销，并为其他优化（如常量传播）创造更多机会。此Pass能处理返回值、多返回点等复杂情况。

*   **循环不变量外提 (Loop Invariant Code Motion, LICM)**: 识别循环中那些每次迭代都计算相同结果的指令（循环不变量），并将它们移动到循环的前置首部（Preheader）中。这显著减少了循环内部的冗余计算，是性能优化的关键一环。

*   **稀疏条件常量传播 (SCCP)**: 一个比普通常量传播更强大的优化。它在CFG上传播常量信息，不仅能进行常量折叠，还能找出不可达的基本块（死代码），从而优化分支指令，将条件分支变为无条件跳转。

*   **强度削减 (Strength Reduction)**: 将高开销的运算指令替换为等价的低开销指令。例如，将乘以2的幂次方的乘法运算替换为更快的左移位运算，或将除以2的幂次方的除法替换为一系列位运算。

*   **死代码消除 (DCE)**: 在其他优化Pass运行后，再次进行清理。它会移除不可达的基本块以及那些计算结果从未被使用过的指令。

### 4.5 后端代码生成器 (`AsmGenerator`)

后端的任务是将优化后的IR转换为目标平台的汇编代码。

*   **栈帧管理**: 在每个函数开始时，`allocStack`会计算所需的栈空间大小（用于保存返回地址、局部变量、数组等），并生成相应的`addi sp, sp, -size`指令（prologue）；在函数结束时，生成恢复栈指针的指令（epilogue）。
*   **寄存器分配**: 实现了一个高效的寄存器分配策略。
    *   **寄存器池**: 维护可用的通用寄存器和浮点寄存器池。
    *   **值映射**: 跟踪每个IR值与物理寄存器之间的映射。
    *   **寄存器溢出 (Spilling)**: 当可用寄存器耗尽时，能够选择一个被占用的寄存器，将其中的值存回栈中（溢出），以便腾出该寄存器给新的值使用。
    *   **调用约定**: 严格遵守RISC-V调用约定，正确处理函数参数传递（通过`a0-a7`, `fa0-fa7`寄存器或栈）和返回值（`a0`/`fa0`）。并在`call`指令前后，自动保存和恢复调用者保存（caller-saved）的寄存器。
*   **指令生成**: `genInstruction`函数是一个巨大的分派中心，它将每一种IR指令（如`Add`, `Load`, `Store`, `Br`等）精确地翻译成一条或多条RISC-V汇编指令。
*   **全局变量与数组**: 正确处理全局变量的定义和初始化。`genArrayAccess`能够根据多维数组的索引计算出正确的内存偏移地址。

## 5. 创新点与技术亮点

1.  **先进的、基于Pass的迭代式优化框架**
    *   `IROptimizer`设计。它不只是简单地实现几个优化算法，而是构建了一个**基于Pass的迭代式优化器框架**。优化器会循环执行所有优化Pass，直到IR代码收敛（不再变化），确保各项优化能够相互促进，达到最大效果。这种设计与LLVM等工业级编译器的设计思想一致，具有极高的模块化和可扩展性。

2.  **高级优化算法的全面实现**
    *   在框架内实现了多个现代编译器中的高级优化算法，覆盖了函数、循环和数据流等多个层面：
        *   **函数内联 (Function Inlining)**: 跨函数边界的优化，能消除调用开销并解锁更多优化可能性。
        *   **循环不变量外提 (LICM)**: 关键的循环优化，能极大提升循环密集型代码的性能。
        *   **Mem2Reg (SSA转换)**: 该算法需要进行复杂的控制流分析（**支配树**和**支配边界**），是后续所有基于数据流分析的优化的基础。
        *   **稀疏条件常量传播 (SCCP)**: 强大的数据流分析，能同时完成常量折叠和死分支消除。

3.  **分层的混合优化策略 (Hybrid AST & IR Optimization)**
    *   本项目没有将所有优化都放在IR层面，而是在AST层就引入了**常量折叠**和**死代码消除**。这种在高级表示（AST）上进行的优化，可以有效简化语法树结构，使得后续生成的IR更加简洁，从而降低IR层面优化的复杂度和执行时间，是一种高效且设计思路清晰的策略。

4.  **完整且健壮的后端实现**
    *   后端的`AsmGenerator`不仅仅是简单的指令翻译，它实现了一个完整的代码生成系统。
    *   **带溢出机制的寄存器分配器**是其亮点。它能够处理寄存器资源紧张的复杂情况，通过将不常用的变量“溢出”到栈上，保证了程序的正确生成，使得编译器能够处理更复杂的函数。
    *   **对RISC-V调用约定的精确实现**，包括函数调用前后对调用者保存寄存器的自动保护和恢复，确保了模块化编程和函数间调用的正确性，这是衡量后端是否健壮的重要标准。

## 6. 如何使用

1.  **克隆项目到本地**
    ```bash
    git clone https://gitlab.eduxiji.net/T202510027205520/compiler2025-hjydypd
    cd compiler2025-hjydypd
    ```

2.  **编译项目**:
    ```bash
    mkdir build
    cd build
    cmake ..
    make
    ```

3.  **运行编译器**:
    ```bash
    # 将SysY源文件编译为汇编文件
    ./compiler -S -o <testcase.s> <testcase.sy>
    ```
4.  **汇编、链接与执行**:
    ```bash
    # 使用RISC-V交叉编译工具链，可使用官方提供的运行时库sysylib，可以选择静态/动态链接 以及 代码的位置相关性
    riscv64-linux-gnu-gcc -o <testcase> <testcase.s> [-L$LIB_PATH -lsysy] [-static] [-fPIC]
    # 使用qemu执行目标文件，请先下载相关工具链
    qemu-riscv64 -L [path_to_riscv64-linux-gnu] <testcase> [< testcase.in] [> testcase.out]
    # 获取返回值
    echo $?
    ```
