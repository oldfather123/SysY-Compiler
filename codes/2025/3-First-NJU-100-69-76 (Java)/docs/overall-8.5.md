# 整体设计

## 基于询问的系统

整个系统中的对象被分为两类，独立存在的实体和依附于实体的属性

对应地存在对指定类型的实体和属性的提供者，它们互相依赖组成了整个体系

在主类中只注册一个输入提供者，并查询最终的输出

## 编译过程

CharStream -> AST -> MIR -> RV64GC LIR -> RV64GC Assembly

解析过程使用ANTLR进行，代码库中带上了antlr4-runtime的代码和许可

AST几乎按原样翻译到MIR上后进行优化，以具有更多约束的标准形式传入Lowering部分，得到RV64GC LIR

在LIR基础上，再做若干优化，并进行寄存器分配（也将部分变量分配到栈上）

在机器代码生成的部分，处理临时寄存器中和栈上的变量，并串行化SSA形式带来的并行移动，翻译到合适的机器代码
