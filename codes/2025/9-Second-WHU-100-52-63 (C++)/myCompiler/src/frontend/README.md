## semantic analysis

### 1. Overview

分析 AST 树是否符合语义规则，主要包括以下几个方面：

1. 有且只有一个 main 函数，main 函数必须是 int 类型，且没有参数
2. 初始化全局变量时，必须是常量表达式
3. 定义数组时，数组下标一定为整数常量表达式，数组元素必须是常量表达式。整型数组初值列表中不能出现浮点型元素。
   初始化器必须是以下三种情况之一：
   (a) 一对花括号 {}，表示所有元素初始为 0。
   (b) 与多维数组中数组维数和各维长度完全对应的初始值，如{{1,2},{3,4},
   {5,6}}、{1,2,3,4,5,6}、{1,2,{3,4},5,6}均可作为 a[3][2]的初始值。
   (c) 如果花括号括起来的列表中的初始值少于数组中对应维的元素个数，则
   该维其余部分将被隐式初始化，需要被隐式初始化的整型元素均初始为
   0，如{{1, 2},{3}, {5}}、{1,2,{3},5}、{{},{3,4},5,6}均可作为 a[3][2]的初
   始值，前两个将a初始化为{{1, 2},{3,0}, {5,0}}，{{},{3,4},5,6}将 a 初始
   化为{{0,0},{3,4},{5,6}}。
4. 初始化列表中的元素个数不能超过数组声明时给出的总的元素个数。
5. 变量/函数声明定义后使用，作用域规则，使用数组是下标一定是整数常量表达式
6. 常量表达式一定是 int 或者 float 类型，中间不能存在 void funcCall
7. 表达式不能使用与或非和比较指令
8. 未显式初始化的全局变量，其（元素）值均被初始化为 0 或 0.0。可以使用
9. Lval 可以为某变量（此时不能出现[]）,也可以为数组元素（此时必须出现[]且维数符合）

数组各维度必须为非负
数组元素初值大小不能超出对应元素数据类型的表示范围
