# 设计文档

## 前言

本文档为 2025 年毕昇杯编译大赛参赛队伍 移进规约都队 的编译器设计文档。本编译器面向官方提供的 sysy 语言，以自研 IR 为中间表示（详见 [IRBuild.md](./IRBuild.md)），以 RISC-V 为最终目标代码的体系结构。

---

## 前端部分

前端详细设计见 [前端设计文档](./Frontend.md)。

- **词法/语法分析**：使用 antlr4 生成解析树（parse tree）。
- **AST 构建**：采用 visitor 模式遍历解析树，生成抽象语法树（AST）。
- **语义分析**：包括类型检查、符号表管理、作用域处理、常量折叠等。
- **设计亮点**：自动化语法分析、AST 结构清晰、符号表多级嵌套、错误报告详细，详见前端文档。

---

## 中间代码生成

IR 设计与实现详见 [IRBuild.md](./IRBuild.md) 和 [IRPass.md](./IRPass.md)。

- **IRBuilder**：递归遍历 AST，生成自研 IR，支持表达式、控制流、数组、函数等。
- **优化 Pass**：包括死代码消除、公共子表达式消除、循环不变代码外提、数组消除等，详见 [IRPass.md](./IRPass.md)。
- **设计亮点**：模块化 IR 结构、SSA 支持、丰富的优化管道、调试友好。

## 后端部分

后端负责将 IR 转换为目标机器代码（RISC-V 汇编），采用经典多阶段设计。

### 后端整体架构

```
IR代码 → 指令选择 → 窥孔优化1 -> 循环优化(LICM) → 寄存器分配 → 窥孔优化2 → 汇编生成 → RISC-V汇编
```

### RISC-V 数据结构

- **`RISCVModule`**: 顶层模块，包含所有函数和全局变量。
- **`RISCVFunction`**: 函数单元，包含基本块序列、栈帧信息和活跃性分析结果。
- **`RISCVBasicBlock`**: 基本块，包含指令序列和控制流信息。
- **`RISCVInstruction`**: 指令表示，支持 R/I/S/B/U/J 型指令和伪指令。
- **`RISCVRegister`**: 寄存器抽象，支持虚拟寄存器和物理寄存器。
- **`RISCVOperand`**: 操作数，支持寄存器、立即数、内存地址和标签。
- **`RISCVGlobalBlock`**: 全局数据块，处理全局变量和常量池。

#### 栈帧设计

栈帧采用标准的 RISC-V ABI 布局：

```
高地址 (函数入口时的 sp)
+------------------+ <- sp (函数入口)
| raStackSize      | ← 保存返回地址 ra 寄存器
+------------------+
| valueStackSize   | ← 局部变量、临时值、alloca等
+------------------+
| argStackSize     | ← 为调用其他函数预留的参数空间
+------------------+ <- sp (函数执行中)
低地址
```

### 指令选择

- **模式匹配**: 根据 IR 指令类型选择对应的 RISC-V 指令模板。
- **操作数转换**: 将 IR 操作数转换为 RISC-V 操作数（寄存器/立即数/内存）。
- **复杂指令分解**: 将复杂操作分解为多条基本指令。
- **ABI 遵循**: 确保函数调用、参数传递符合 RISC-V ABI 规范。

主要处理的指令类型：

- 算术运算（add, sub, mul, div 等）
- 逻辑运算（and, or, xor, shift 等）
- 内存访问（load, store）
- 控制流（branch, jump, call, ret）
- 浮点运算（fadd, fsub, fmul, fdiv 等）
- 类型转换（int↔float）

### 窥孔优化

窥孔优化（Peephole Optimization）是一种局部代码优化技术，通过分析小窗口内的指令序列来识别和消除冗余、低效的指令模式。

#### 架构设计

窥孔优化采用了可扩展的 Pass 管理架构：

- **`PeepOptimizationManager`**: 优化管理器，协调多个优化 Pass 的执行
- **`PeepPass`**: 抽象优化 Pass 基类，定义了统一的优化接口
- **模块化设计**: 每个具体的优化策略都实现为独立的 Pass，便于维护和扩展

#### 优化策略

目前实现的窥孔优化包括：

1. **冗余指令消除**

   - 消除连续的相同指令
   - 消除无效的移动指令（`mv rd, rd`）
   - 删除多余的 jal 指令

2. **指令强度削弱**
   - 将乘法指令转化为位移指令

### 循环不变量代码外提（LICM）

循环不变量代码外提是一种重要的循环优化技术，将循环内不依赖于循环变量的计算移到循环外部，减少重复计算开销。

#### 核心思想

在循环执行过程中，某些指令的结果在每次迭代中都相同（如数组基地址计算），这些指令可以移动到循环的 preheader 中执行一次，而不是在每次迭代中重复执行。

#### 算法实现

**1. 循环分析**

```cpp
void LICM::analyseLoops()
```

- 识别函数中的所有顶层循环
- 构建循环嵌套关系
- 为每个循环确定 preheader 基本块

**2. 不变量识别**

```cpp
bool LICM::isLoopInvariant(shared_ptr<RISCVInstruction> inst)
```

当前主要处理 LA（Load Address）指令：

- LA 指令用于加载全局变量或常量的地址
- 这些地址在循环执行过程中保持不变
- 适合外提到循环 preheader

**3. 指令依赖分析**

```cpp
unordered_map<...> LICM::getInvariantMap(shared_ptr<RISCVLoop> loop)
```

- 扫描循环内的所有基本块
- 识别与 LA 指令相关的后续指令（ADD、ADDI、Load、Store 等）
- 建立 LA 指令与其依赖指令的映射关系

**4. 相同指令合并**

使用自定义哈希和比较函数实现语义等价的 LA 指令合并：

```cpp
struct InstructionHash {
    // 基于操作码和标签计算哈希，忽略目标寄存器差异
}

struct InstructionEqual {
    // 对于LA指令，只比较标签，忽略目标寄存器
    // 实现 "la t0, array" 和 "la t1, array" 的语义等价性
}
```

**5. 指令外提与寄存器合并**

```cpp
void LICM::insertLAInst(...)     // 将LA指令插入preheader
void LICM::mergeLAReg(...)       // 统一相关指令的寄存器使用
```

#### 优化效果

- **减少重复计算**: 将 O(n)次的地址计算减少到 O(1)次
- **提高缓存效率**: 减少指令 cache 的压力
- **支持嵌套循环**: 递归处理多层循环嵌套
- **寄存器优化**: 通过合并减少寄存器使用压力

#### 处理挑战

1. **指令依赖**: 准确识别与 LA 指令相关的后续指令
2. **寄存器重命名**: 合并时需要统一不同 LA 指令使用的寄存器
3. **循环嵌套**: 正确处理父子循环间的外提关系
4. **代码稳定性**: 使用内容哈希确保优化结果的确定性

### 寄存器分配

- **活跃性分析**：`LivenessInfo`类提供精确的活跃区间、def-use 链、冲突检测、溢出决策等。
- **图着色算法**：基于《虎书》的经典算法，包含以下阶段：

1. **BUILD**: 构建冲突图和移动指令图

   - 活跃性分析计算寄存器生命周期
   - 构建寄存器冲突关系
   - 识别可合并的移动指令

2. **SIMPLIFY**: 简化低度数节点

   - 移除度数小于 K 的节点（K 为可用寄存器数）
   - 将节点压入栈中

3. **COALESCE**: 合并移动相关的寄存器

   - Briggs 保守合并策略避免增加着色复杂度
   - 减少不必要的移动指令

4. **FREEZE**: 冻结移动指令

   - 当无法继续合并时，放弃部分移动指令的合并

5. **SPILL**: 处理溢出

   - 选择溢出代价最小的寄存器
   - 生成 load/store 指令访问内存

6. **SELECT**: 实际着色
   - 从栈中弹出节点并分配颜色
   - 处理溢出寄存器的重写，自动插入 load/store 指令，保证正确性。

### 代码生成

- **全局数据段**: 生成 .data 段的全局变量和常量。
- **函数序言/尾声**: 生成栈帧建立与恢复代码。
- **指令序列**: 将 RISC-V 指令转换为汇编文本。
- **符号解析**: 处理标签和跳转目标。

### 后端优化亮点

1. **循环不变量代码外提（LICM）**

   - 智能识别循环内的不变量计算（主要是 LA 指令）
   - 支持嵌套循环的递归外提
   - 基于内容的指令合并，避免冗余的地址计算
   - 自定义哈希函数确保优化结果的确定性

2. **窥孔优化**
   - 可扩展的 Pass 管理架构
   - 模块化的优化策略设计
   - 在寄存器分配后执行，获得最佳优化效果
   - 支持多种指令序列的模式匹配和替换
